# 模板系统

模板是 Amux Studio 的核心能力之一。通过模板，创作者可以将经过验证的提示词结构沉淀下来，实现高效复用。

## 什么是模板

模板由以下部分组成：

| 字段 | 说明 |
|------|------|
| **标题** | 模板名称，如"电影级人像写真" |
| **分类** | 所属分类：人像、产品、风景、抽象等 |
| **提示词** | 核心 Prompt，支持 `{{变量}}` 语法 |
| **变量定义** | 变量名、类型、默认值、可选值 |
| **封面图** | 模板效果展示图 |

## 变量语法

模板提示词中使用 `{{变量名}}` 标记可替换的部分：

```
A cinematic portrait of a {{gender}}, {{age_range}} years old,
with a {{expression}} expression. Shot in {{lighting}} lighting
with {{color_tone}} color grading.
```

用户使用模板时，只需填写各变量的值即可生成完整的提示词。

## 模板状态流转

```
创建(PENDING) → 提交审核(IN_REVIEW) → 通过(APPROVED) / 拒绝(REJECTED)
                                         ↓
                                      已归档(ARCHIVED)
```

- **PENDING** — 草稿状态，仅作者可见
- **IN_REVIEW** — 提交审核，等待管理员审核
- **APPROVED** — 审核通过，在模板市场公开展示
- **REJECTED** — 审核未通过，可修改后重新提交
- **ARCHIVED** — 已归档，从市场下架但作者仍可访问

## 创建模板

1. 进入 **我的模板** 页面
2. 点击 **创建模板**
3. 填写模板信息和提示词
4. 定义变量参数
5. 上传效果封面图
6. 保存后提交审核

## 使用模板

1. 在 **模板市场** 浏览或搜索模板
2. 点击模板查看详情
3. 填写变量参数
4. 选择生成模型
5. 点击生成

## 最佳实践

- **提示词结构化**：按照主体、场景、风格、技术参数的顺序组织
- **变量粒度适中**：太细碎增加填写成本，太粗放失去灵活性
- **提供默认值**：为变量设置合理的默认值，降低使用门槛
- **附带优秀案例**：封面图展示最佳效果，吸引更多用户使用
